01 QUEST

安装之前，先问这几个问题

很多 AI 模组只会刷屏。Narrator-o 想解决的是：旁白如何成为玩法，而不是噪音。

Q01

挖了三小时矿，谁来点评这场灾难？

没有观众时，是否希望有人把「又掉进岩浆」钉在耻辱柱上？

Q02

旁白该不该动手改世界？

只说话很安全；能切模式、找群系、砸人一记才像节目效果——如何不卡服？

Q03

中文字幕、英文语音如何同时成立？

旁白能否按玩家语言给字幕，同时走稳定的语音通道？

Q04

API 密钥该不该进客户端？

不该。密钥只在服务端；玩家只消费旁白结果。

Q05

安静观察者，还是疯批主持人？

整活强度能否一旋钮调节，而不是改 Prompt 玄学？

Q06

喊旁白时，旧台词该不该被打断？

/nsay 应像被点名——立刻转向，而不是背完旧稿。

如果这些问题戳中你：观察 → 检索技能 → LLM 出计划 → 技能/说话/搜索 → 字幕与语音落地。这是服务端导演系统，不是聊天机器人贴皮。
02 CRAFT

从想法落到能玩的六步

面向服主与单人档主人。

1

安装

Minecraft 1.21.1 + NeoForge，放入 narratoro-0.1.0-alpha.jar。AI 调用只在服务端。

2

接通 LLM / TTS

游戏内按 ] 打开配置，或：

/narratoro setup llm <base_url> <model> <api_key>
/narratoro setup tts <base_url> <model> <api_key> [voice]
/narratoro config save

密钥不回显。改完务必 config save。仅配 LLM 也应能出字幕；配齐 TTS 才有语音。

3

冒烟测试

/narratoro status
/narratoro test speak <玩家> 你好，世界。
4

点名旁白

/nsay 评价一下我这套装备
/nsay 找沙漠并把我传过去

会打断旧旁白队列，优先响应你。

5

整活强度

persona.chaos 0–10：低克制、中约一半吐槽带技能、高更疯。

6

玩家体验

语音开关、音量、字幕位置/大小；/narratoro player language zh_cn 作语言回退。
